| Vo Hai Dung (Team Leader, Aimer PAM) | **What went well.** The Work Breakdown Structure was followed end-to-end: every leaf task has a corresponding script, configuration file, or report section, which kept daily progress measurable. Centralising the random seed in ``src.utils.repro`` and the configuration schema in ``src.configs.schema.ExperimentConfig`` made every reported number reproducible from a single phase script. Reporting Expected Calibration Error, latency, and per-class error analysis alongside accuracy lifted the evaluation noticeably above a single F1-macro number. **What was difficult.** Three recurring environment issues consumed budget: (i) the system-managed ``blinker`` package on Ubuntu 24.04 blocked ``pip install -e .`` until ``--ignore-installed blinker`` was passed; (ii) the latest ``torch 2.11+cu130`` on PyPI was built against a CUDA toolkit newer than the host driver, forcing a downgrade to ``torch 2.6.0+cu124``; (iii) the ``DeBERTa-v3`` tokeniser requires ``sentencepiece`` installed separately, and ``LogisticRegression(solver="liblinear")`` is binary-only in modern scikit-learn so the multinomial ``"lbfgs"`` had to be used instead. Translating 120,000 articles through OPUS-MT on a single GPU also required reducing the beam search from 8 to 4 and batching by 32 examples to fit the time budget. **What I would change next time.** Pin ``torch`` to a specific CUDA wheel in ``requirements.txt`` from the first commit, ship a minimal ``Dockerfile`` so the environment is reproducible without manual workarounds, and start the calibration / latency benchmarks as soon as Phase 4 finishes rather than waiting until Phase 7 so the gap to O5 surfaces earlier in the schedule. **Skills developed.** Hugging Face Trainer fine-tuning with bf16 and label smoothing; custom ``Trainer`` subclassing (R-Drop); cosine schedule with linear warm-up; OPUS-MT translation and back-translation augmentation; SetFit data-efficiency curves; SHAP TextExplainer and LIME for explanation; class-wise isotonic and temperature-scaling calibration with reliability diagrams; ONNX export and dynamic INT8 quantisation; latency-accuracy Pareto-front benchmarking; multi-model Gradio deployment. **Response to the instructor's Action Plan recommendations.** (1) Reproducibility — ``requirements.txt`` pins all 25 runtime dependencies and ``src.utils.repro.set_seed`` is the single seed entry-point. (2) Vietnamese back-translation quality — Section 3.3.3 reports the full 2×2 ablation per language (encoder × back-translation) on all four cells; translation quality was not measured against a human-validated reference and is explicitly listed as a scope-out in Section 4.1.4. (3) Calibration and latency reported alongside accuracy — Section 3.5.2 reports a four-calibrator post-hoc grid (baseline / temperature / isotonic / chained T → isotonic) with reliability diagrams; Section 3.5.3 reports a twelve-model latency-accuracy Pareto front at batch 1, both single-thread and multi-thread. (4) Small / base runs as ablation baselines — the four-row table in Section 3.3.2 explicitly labels small / base as ablation and base / large as primary; a follow-up 2 × 3 variant × seed grid (Section 3.3.2.5) plus a 3-seed soft-voting ensemble closes the residual gap to O1 (F1 = 0.9528). (5) Document deviations openly — Section 3.5.2 documents that O4 was met only after introducing class-wise isotonic regression (temperature scaling alone fell short because the trained model is under-confident due to label smoothing); Section 3.5.3 documents that O5 is unmet on single-stream CPU on the benchmarked host; the redirect of O6 from a Hugging Face Space to a public GitHub repository is recorded in Section 1.2.2 rather than backported into the Action Plan. **Acknowledgement.** Thanks to Ths. Nguyen Trung Hau (instructor, Samsung Innovation Campus) for the Action Plan review and the focus-area recommendations that shaped the final pipeline; to the open-source maintainers of the Hugging Face Transformers, Datasets, and Optimum libraries, the OPUS-MT project, Cleanlab, BERTopic, SHAP, LIME, ONNX Runtime, Optuna, scikit-learn, PyTorch, and Gradio for the toolkit that this project relies on; and to the Samsung Innovation Campus for the AI Course curriculum and the capstone project framework. |
